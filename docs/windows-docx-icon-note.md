# 导出 docx 属性正常，但图标显示为「白页」的说明

## 原因说明

- **文件属性**（作者、标题、程序名称等）来自 Word 包内的 `docProps/core.xml`、`docProps/app.xml`，与资源管理器**图标**是两套机制。
- **图标**由 Windows 根据 **扩展名 `.docx`** 和 **注册表里的 ProgID**（如 `Word.Document.12`）从系统图标资源里取，**不会**读你在属性里写的「应用程序」字符串来换图标。
- 因此：**元数据已改对，但图标仍不对**，多半是 **Windows 壳层（缓存 / 安全标记 / 刷新）** 问题，而不是 docx 内容不合法。

## 常见情况

1. **从浏览器下载到桌面**  
   文件可能带 **Mark of the Web**（「来自其他计算机」）。部分环境下会短暂出现**通用文档图标**，或需刷新后才显示 Word 图标。  
   - 在文件 **属性 → 常规** 若底部有 **「解除锁定」**，勾选后确定，再看重图标是否恢复。

2. **图标缓存**  
   旧缓存仍显示白页图标。可尝试：  
   - 当前文件夹按 **F5** 刷新；  
   - 注销/重启；  
   - 管理员 PowerShell 执行（Win10/11 视版本而定）：  
     `ie4uinit.exe -ClearIconCache`  
     或重启 **Windows 资源管理器**（任务管理器 → Windows 资源管理器 → 重新启动）。

3. **扩展名被隐藏导致误判**  
   若开启「隐藏已知文件类型的扩展名」，实际可能是 `export_xxx.docx.txt` 之类，图标会不对。  
   - 在文件夹选项 → 查看 → **取消**勾选「隐藏已知文件类型的扩展名」，确认文件名以 **`.docx`** 结尾。

4. **验证文件本身**  
   合法 docx 本质是 **ZIP**，文件头应为 **`PK`**（用十六进制查看器看前 2 字节）。  
   - 若 **Word 能正常打开**，且扩展名是 `.docx`，一般即为合法文件，图标问题可按上面系统侧步骤处理。

## 与项目代码的关系

- 服务端已使用 **正确的 MIME**（`application/vnd.openxmlformats-officedocument.wordprocessingml.document`）和 **`.docx` 文件名** 导出；**无法在服务器生成的 docx 里写入「强制 Windows 显示 Word 图标」的字段**——这是操作系统壳层行为。
- 若 **同一台电脑** 上 **`1111.docx` 是蓝 W 图标、导出的 `export_*.docx` 是白页**，优先按 **解除锁定 + 刷新图标缓存** 排查；二者扩展名相同时，注册表关联应一致。
